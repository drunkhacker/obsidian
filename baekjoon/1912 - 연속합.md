i번째 왔을때 부분합의 마지막 index가 i-1인가?
a(i)를 넣고 작아진다? -> 버린다. 즉, 마지막 index 변경 없음. 부분합도 변경 없음
a(i)를 넣고 커진다? -> 최대 부분합에 a(i)더한다, 마지막 index를 i로 바꾼다. 

i번째 왔을때 부분합의 마지막 index가 i-1보다 적은가? (즉 이미 연속이 아닌가?)
그럼 이때는 기존 최대합이 계속 유지되어 다음 a(i+1)로 넘어갈지 아니면 a(i)부터 새 부분합을 시작하는게 좋은지 판단해야함
a(i) < 부분합? 버려.  넘어감
a(i) >= 부분합? 리셋. M = a(i). 마지막 인덱스 = i

초기시도.
```
#include <stdio.h>

int a[100001] = {0, };
int main() {
    int n, i;
    int I, M;

    scanf("%d", &n);
    for (i=0; i<n; i++) {
        scanf("%d", &a[i]);
    }

    I = 0;
    M = a[I];
    for (i=1; i<n; i++) {
        if (I == i-1) {
            if (a[i] >= 0) {
                M += a[i];
                I = i;
            }
        } else {
            if (a[i] >= M) {
                M = a[i];
                I = i;
            }
        }
    }

    printf("%d\n", M);

    return 0;
}

```

다시생각해보면 마지막 index가 i-1 번째였을때, a(i)가 0보다 작아도 계속 킵해놓는게.. 나을수 있음. M+a(i)가 0보다 작지만 않으면!

i번째 뒤에를 봐야하는 상황임. 나까지만 보면 안됨..음수일때만 그런거 아닐까?

아예 새로운 접근..
- n번째 수부터 새로 부분합을 시작할때, (= an)
- 그전에 있었던 부분합에 그냥 an 더할때의 값 (= acc)
	- 기존에 있었던 최대 부분합 (= M)
을 두고 각 단계에서 an이 acc보다 크면 acc를 업데이트, acc가 M보다 크면 다시 M을 업데이트 하는 방식으로 for loop을 돌려보자

```
#include <stdio.h>

int a[100001] = {0, };
int main() {
    int n, i;
    int acc, M;

    scanf("%d", &n);
    for (i=0; i<n; i++) {
        scanf("%d", &a[i]);
    }

    M = acc = a[0];
    for (i=1; i<n; i++) {
        if (acc + a[i] < a[i]) {
            acc = a[i];
        } else {
            acc += a[i];
        }

        if (acc > M) {
            M = acc;
        }
    }

    printf("%d\n", M);

    return 0;
}

```
