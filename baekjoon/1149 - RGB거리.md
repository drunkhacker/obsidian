https://www.acmicpc.net/problem/1149

i번째에서 i-1번째 color가 없는 것으로 칠해보고 결과를 따진다.
즉, i번째까지 집을 칠할때 최소 비용은 min(m\[i-1\] + ri + f(r, i+1, . .)) 인데 i+1번째부터 다시 무슨 색을 칠할지 알아야 i번째까지 집을 칠할때 최소 비용을 안다? 는 이상하니까.. 

dp 배열을 돌릴때 i와 color 2dimension으로 돌리자.
dp\[i\]\[c\] = i번째를 c로 칠할때 0~ i번째까지 집을 칠할때 든 최소 비용

그렇다면 i번째에는 
dp\ii\]\[R\] = min(dp\ii-1\]\[G\], dp\ii-1\]\[B\])  + Ri
dp\ii\]\[G\] = min(dp\ii-1\]\[R\], dp\ii-1\]\[B\])  + Gi
dp\ii\]\[B\] = min(dp\ii-1\]\[G\], dp\ii-1\]\[R\])  + Bi

dp\[0\]\[R\] = Ri
dp\[0\]\[G\] = Gi
dp\[0\]\[B\] = Bi

```c
#include <stdio.h>

#define R 0
#define G 1
#define B 2
#define min(a,b) ((a) > (b) ? (b):(a))

int dp[1001][3] = {0, };
int c[1001][3] = {0, };

int main() {
    int N, i, j;
    scanf("%d", &N);

    for (i=0; i<N; i++) {
        for (j=0; j<3; j++) {
            scanf("%d", &c[i][j]);
        }
    }

    dp[0][R] = c[0][R];
    dp[0][G] = c[0][G];
    dp[0][B] = c[0][B];

    for (i=1; i<N; i++) {
        dp[i][R] = min(dp[i-1][G], dp[i-1][B]) + c[i][R];
        dp[i][G] = min(dp[i-1][R], dp[i-1][B]) + c[i][G];
        dp[i][B] = min(dp[i-1][R], dp[i-1][G]) + c[i][B];
    }

    printf("%d\n", min(min(dp[N-1][R], dp[N-1][G]), dp[N-1][B]));
    return 0;
}

```
