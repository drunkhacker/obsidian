N이 계속 줄어드는데, 특정 값이 됐으면 그 값에서 1로 만드는데 드는 최소 연산 수는 이미 고정. -> memoization

3으로 나누던지, 1을 빼던지
2로 나누던지, 1을 빼던지
이도저도 아니면 1을 뺀다 ?

dp\[i\] = i를 1로 만드는데 드는 최소 연산 수 
dp\[i\] = 
- if i % 3 == 0:
	- min(dp\[i/3\], dp\[i-1\]) + 1
- else if i%2 == 0:
	- min(dp\[i/2\], dp(\[i-1\]) + 1
- else
	- dp\[i-1\] + 1


위 식은 틀렸고.. 왜냐면 6같은 경우는 2의 배수이기도 하고 3의 배수이기도 하기때문에, else if 로 거르면 안됨.  다 따져봐야하는 케이스가 있어서 그걸 고려해야함.

```c
#include <stdio.h>
#define min(a,b) ((a)>(b)?(b):(a))

int dp[1000001] = {0,};

int main() {
    int N, i;
    scanf("%d", &N);

    dp[1] = 0;
    dp[2] = 1;
    for (i=3; i<=N; i++) {
        dp[i] = dp[i-1] + 1;
        if (i%3 == 0) {
            dp[i] = min(dp[i/3] + 1, dp[i]);
        }

        if (i%2 == 0) {
            dp[i] = min(dp[i/2] + 1, dp[i]);
        }
    }

    printf("%d\n", dp[N]);

    return 0;
}

```
