들어온 그래프를 트리로 보고, 트리의 leaf node들을 모은 다음에, 그 집합에서 노드 두개로 이루어진 순서쌍 (u, v)을 뽑아낸다.

u, v의 최소깊이 공통조상을 L = LCA(u,v) 이라고 하면, u, v 사이의 거리는 D(u, L) + D(v, L) 이 된다. D(u, L)은 L을 만날때까지 부모를 따라가면서 간선 가중치를 더하면 된다.

이 과정을 leaf node로 이뤄진 모든 (u, v) 순서쌍에 대해 돌리면서 최대값을 찾으면 그게 트리의 지름.

LCA를 찾는 알고리즘은 https://www.crocus.co.kr/660 를 참고해보자.

그래프 저장형태는 adj list로.

LCA 와꾸:
`ac[v][i]` = v의 $2^{i}$번째 조상 이라고 세팅.
$2^{i-1}$번씩 두번가면 $2^{i}$ 이므로, v의 $2^i$ 번째 조상은 v의 $2^{i-1}$번째 조상에서 다시 $2^{i-1}$ 번째 조상과 같다.
`ac[v][i] = ac[ac[v][i-1]][i-1]`

`ac[]`배열을 populate 하려면, dfs를 하면서 내려가는데, u에서 v로 방문했을때, `ac[v][0] = u` 이다.
```
ac[v][1] = ac[ac[v][0]][0]
ac[v][2] = ac[ac[v][1]][1]
...

루트 나올때까지 돌리면 되지 않을까?
```

`ac[][]`배열의 두번째 인자 깊이는 100000보다 약간 크게.. $2^{17}$ 정도

leaf들만 다 잡고 순서쌍으로 돌리는 방법으로 하니까 timeout이 났음.

트리의 지름을 구하는 알고리즘은 다음과 같다.
참고 https://blog.myungwoo.kr/112

트리 내의 임의 정점 $x$ 에서 가장 먼 정점 $y$ 를 찾고, 그 $y$에서 다시 가장 먼 정점 $z$를 찾는다. $y$와 $z$사이의 거리가 트리의 지름이다.

증명) 귀류법 이용.
가장 거리가 긴 정점 u, v 가 있다고 하자. 즉 d(u,v)가 지름. 트리의 임의 정점 x에서 가장 먼 정점 y를 구했다고 할때 경우의 수가 다음과 같이 나뉜다. 

1. x가 u나 v: x에서 가장 먼 정점을 구하면 v혹은 u로 떨어질거고, 다시 u,v에서 가장 먼 정점을 구하면 v, u로 떨어질거므로 성립한다.
2. y가 u나 v: y에서 가장 먼 정점을 구하면 v, u 일 것이고 그러면 알고리즘 성립
3. x, y, u, v가 다 다른 경우
1) x에서 y로 가는 패스 x ~ y가 u ~ v와 정점 t를 공유한다고 해보자.
```

  ┌───┐                ┌───┐
  │ x ├─────────┐      │ y │
  └───┘         │      ├───┘
                │      │
                │      │
               t│      │
      ┌─────────.──────┘
      │         │
  ┌───┤         ├───┐
  │ u │         │ v │
  └───┘         └───┘

```
 d(x, y) = d(x, t) + d(t, y)이다. 근데 x에서 가장 먼 정점이 y이므로, d(x, u) = d(x, t) + d(t, u)나 d(x, v) = d(x, t) + d(t, v)가 d(x, y)보다 작으므로, 
 d(t, y) > d(t, u)  AND d(t, y) > d(t, v) 이다.
 따라서 d(u, v) = d(u, t) + d(t, v) < d(u, t) + d(t, y) 가 된다. 근데 아까 d(u, v)가 최대 정점거리라고 했으니 이는 모순이다. 

2) x~y가 u~v 와 공통으로 지나는 정점이 없다.
```
┌───┐                ┌───┐
│ x ├──d3───.────d4──┤ y │
└───┘       │t1      └───┘
            │
            │
            d
            │
   ┌───┐    │    ┌───┐
   │ u ├─d1─.─d2─┤ v │
   └───┘   t2    └───┘
```

위와 같은 케이스에서 x 에서 가장 먼 정점이 y이므로, 
d(x, v) = d(x, t1) + d(t1, t2) + d(t2, v) < d(x, t1) + d(t1, y). 즉, d + d2 < d4, d2 < d4.
d >= 0 이므로, d(u, v) = d1 + d2 < d1 + d + d4 = d(u, y). 따라서, 이는 d(u, v) 가 지름이라는 것에 모순.

임의 정점 u로부터 가장 먼 정점 v를 찾는 방법:
dfs로 탐색을 하는데, 각 단계마다 parent와 그때까지 내려온 거리 합, parent와 현재 vertex의 거리를 가지고 가면서 거리 합에 거리를 누적시키면서 검사해보면 될듯.


```c
#include <stdio.h>
#include <stdlib.h>

#if 0
#define write_log(format,args...)  printf(format, ## args)
#else
#define write_log(format,args...)
#endif

#define MAX_V 100000

typedef struct node {
    int weight;
    int vertex;
    struct node *next;
} Node;

typedef struct {
    Node *adj[MAX_V+1];
    Node *tail[MAX_V+1];
} Graph;

Graph g;

void add_edge(int u, int v, int w) {
    Node *node = malloc(sizeof(Node));
    node->next = NULL;
    node->weight = w;
    node->vertex = v;

    if (!g.adj[u]) {
        g.adj[u] = node;
        g.tail[u] = node;
    } else {
        // find tail
        Node *tail = g.tail[u];
        tail->next = node;
        g.tail[u] = node;
    }
}

int nv, mv, md;

void dfs(int u, int v, int ws) {
    // u parent
    // v current
    // w = weight(u, v)
    // ws = weight sum acc.

    if (ws > md) {
        md = ws;
        mv = v;
    }

    Node *node = g.adj[v];
    while (node) {
        if (node->vertex != u) {
            dfs(v, node->vertex, ws + node->weight);
        }
        node = node->next;
    }
}

int main() {
    int u, v, w;
    scanf("%d", &nv);
    for (int i=0; i<nv; i++) {
        scanf("%d", &u);
            write_log("u=%d\n", u);
        while (1) {
            scanf("%d", &v);
            if (v == -1) {
                break;
            }
            scanf("%d", &w);
            write_log("v=%d, w=%d\n", v, w);
            add_edge(u, v, w);
        }
    }


    dfs(0, 1, 0);
    dfs(0, mv, 0);

    printf("%d\n", md);

    return 0;
}
```
