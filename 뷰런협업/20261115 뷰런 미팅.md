## 본 미팅 요약
### 1. 프로젝트 목적 및 방향

- 단기적으로는 **자율주행 학습용 원시 비전 데이터셋 구축**이 1차 목표.
    
- 장기적으로는:
    
    - 로보택시/자율주행 시대에 대비한 **라이드헤일링 플랫폼의 전략적 포지셔닝**
        
    - 데이터 소유권, 보상 구조, 블록체인 기반 데이터 권리 모델까지 염두.
        
- 단발성 PoC가 아니라, 향후 **자율주행 인프라·운영·데이터 비즈니스로 확장 가능한 탐색 프로젝트** 성격.

---

### 2. 센서 및 차량 구성

### 카메라 구성

- 기본 구성:
    
    - 전방 3, 측후방 포함 총 **5~6대 카메라**
        
    - 해상도: **최소 FHD (1920x1080) 이상**
        
    - 포맷: PNG (데이터 용량 매우 큼)
        
- 스테레오 / LiDAR:
    
    - 필수는 아님.
        
    - 최신 자율주행 트렌드는 **Mono Vision + DNN 기반 Depth 추정**.
        
    - 필요 시 RTK, LiDAR, Stereo 추가 가능.

### 차량 신호 (CAN / 조작 정보)

- 필수 항목:
    
    - Steering, Brake, Accelerator 입력
        
- 문제점:
    
    - 싱가포르 차량은 CAN 접근 가능 여부 불확실
        
    - 필요 시 리버스엔지니어링 업체 활용 가능 (추가 비용 발생)

---

### 3. 데이터 저장 및 전송 방식

### 온라인 전송

- 현실적으로 불가능:
    
    - 대용량 이미지 → 통신 대역폭, 비용, 안정성 문제
        
- 결론: **오프라인 수거 방식 필수**
    

### 오프라인 파이프라인 구조 (실제 운영 사례)

- 차량 내부:
    
    - PC + 내부 SSD에 실시간 저장
        
    - 백그라운드 프로세스가 일정 주기로 외장 SSD로 복사
        
- 운행 종료:
    
    - 외장 SSD 분리 → 스테이션에 물리적 수거
        
    - 업로드 완료 후 재배포
        
- 안정성 이슈:
    
    - 주행 중 전원 차단, 파일 손상 방지 위해 **전날 데이터만 이동**하는 구조 사용

---

### 4. 싱가포르 법·규제 이슈

- 싱가포르 PDPA(개인정보보호법) 적용 대상.
    
- 핵심 포인트:
    
    - **자율주행 운행 허가와 단순 데이터 수집은 별개**
        
    - 그러나 **비식별화(Blur, Anonymization)는 필수**
        
    - 차량 외부에 촬영 중임을 고지하는 스티커 필요 가능성
        
- 기술적으로:
    
    - 실시간 또는 오프라인 **얼굴·번호판 블러링 파이프라인 구축 가능**
        
- LTA가 자율주행에 적극적이므로:
    
    - 연구 목적 데이터 수집에 대해 **규제 특례 가능성 존재** (추가 확인 예정)
        

---

### 5. RTK / 위치 정밀도

- RTK 필요 시:
    
    - 보정 신호 수신용 통신 모듈 필요
        
    - 상용 RTK 서비스 **월 구독 비용 발생**
            

---

### 6. 차량 대수 및 스케일

- 데이터셋 구축 단계:
    
    - **초기 1대 → 확장 시 최대 10대 내외면 충분**
        
- 실제 상용 자율주행 서비스 단계:
    
    - 센서 구성 자체가 완전히 달라지므로 본 프로젝트와 별도 트랙.
        

---

### 7. 구축 비용 및 역할 분담

- 구성 요소:
    
    - 차량 내 PC 마운트
        
    - 카메라 리그
        
    - 저장 스테이션
        
    - RTK 모듈
        
    - CAN 해킹/연동
        
- 대략적 인식:
    
    - 차량 1대당 **수천만 원 단위 구축비 가능성**
        
- 향후 정리 필요:
    
    - 구축비 분담 구조
        
    - 데이터 소유권 (공동 vs 분리)
        
    - 장기 수익 모델 연계 (자율주행, 데이터 비즈니스)
    
---
